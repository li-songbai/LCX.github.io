一、装饰者模式

动态地给一个对象添加一些额外地职责。就增加功能来说，Decorator模式相比生成子类来说更为灵活。

适用性

  • 在不影响其他对象的情况下，以动态透明的方式给单个对象增加职责。
  • 处理那些可以撤销的职责。
  • 当不能采用生成子类的方法进行扩充时。

参与者

  • Component

    定义一个对象接口，可以给这些对象动态地添加职责。

  • ConcereteComponent

    定义一个对象，可以给这个对象添加一些职责。

  • Decorator

    维持一个指向Component对象地指针，并定义一个与Component接口一致地接口

  • ConcreteDecorator

    向组件添加职责。
